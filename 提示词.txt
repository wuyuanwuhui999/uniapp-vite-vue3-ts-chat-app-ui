# 开发规范（必须严格遵守）

## 一、全局样式与设计规范 (Theme)
除了ChatPage页面，其他页面每个模块使用卡片式显示，卡片设置白色背景+圆角+内边距，所有样式实现必须引用 `theme/` 目录下的变量，严禁硬编码（直接写死具体数值或颜色）。

### 1. 间距规范 (Spacing)
- **统一间距**：所有 `padding`、`margin`，以及图标之间、按钮之间、图文之间、布局之间、文字到边缘、列表项上下的间距，**必须**使用 `theme/size.less` 中的 `@middle-padding`。
- **其他尺寸**：
    - 按钮高度：`@btn-height`
    - 按钮/卡片圆角：`@btn-border-radius`
    - 输入框高度：`Dimens.inputHeight`，其圆角为 `inputHeight / 2`
    - 卡片圆角：`Dimens.borderRadius`

### 2. 色彩规范 (Color)
所有颜色**必须**引用 `theme/color.less` 中的变量。
- **页面背景**：`@page-background-color`
- **主题主色调**：`@primary-color` (应用于主按钮、激活状态)
- **警告/删除色**：`@warn-color` (应用于删除按钮背景)
- **确认按钮**：背景色 `@primary-color`，文字色 `@white-color`
- **取消按钮**：背景色透明，边框色和文字色为 `@gray-color`
- **禁用按钮**：背景色统一为 `@gray-color`

### 3. 图标与字体规范 (Icons & Typography)
- **图标尺寸**：
    - 小图标：`@small-icon-size`
    - 中等图标：`@middle-icon-size`
    - 大图标：`@big-icon-size`
- **图标透明度**：统一设置为 `0.5`
- **字体大小**：
    - 正文/正常文字：`@normal-font-size`
    - 大标题：`@big-font-size`

## 二、代码与架构规范 (Code & Architecture)

### 1. 文件组织 (Structure)
- **页面组件**：放在 `pages/` 文件夹。
- **可复用组件**：放在 `components/` 文件夹。
- **类型定义**：所有 `interface` 或 `type` 统一放在 `types/index.ts`。
- **工具函数**：放在 `utils/` 目录。
- **后端请求方法**：全部封装在 `service/index.ts` 中。
- **API 地址常量**：全部定义在 `api/index.ts` 中。
- **路由配置**：放在 `routers/index.ts`。
- **状态管理**：放在 `stores/useStore.ts`。

### 2. 代码编写要求 (Implementation)
- **注释**：为每个方法（Method）和主要的业务逻辑添加清晰的中文注释。
- **网络请求**：
    - 除登录接口外，所有请求的 `Header` 中必须添加 `Authorization: Bearer token`。
    - 所有请求方法（GET/POST）都在 `service/index.ts` 中实现。
- **修改说明**：当你修改或输出代码后，请在代码块外**用文字说明你修改了哪个文件、哪个方法或哪一部分**。

## 三、数据与接口规范 (Data & API)

### 1. 接口定义 (API Definition)
所有后端接口地址统一以字符串常量形式定义在 `api/index.ts` 中。

### 2. 后端返回数据格式 (Response Format)
所有接口返回的数据结构必须遵循 `HttpUtils/MyAwesomeData` 格式：
```typescript
interface MyAwesomeData<T = any> {
  status: string;      // "SUCCESS" 表示成功，其他值表示失败
  data: T;             // 具体数据，可以是 JSON、数组、字符串、数字
  total: number | null; // 仅分页接口返回总条数，非分页接口返回 null
  msg: string;         // 服务器返回信息，通常仅在失败时存在
}
```


## 三. 数据类型定义 (Types)

- **所有涉及的对象类型、接口返回类型、请求参数类型，都必须在 types/index.ts 中定义并使用。**
- **类型命名规范**：
    - **请求参数类型**：XXXRequest
    - **响应数据类型**：XXXResponse
    - **实体类型**：XXXEntity
    - 必须为每个字段添加注释说明其含义。

## 四、输出格式要求 (Output Format)
    - 小文件输出：如果修改后或生成的单个文件总代码行数少于 500 行，请输出该文件的完整代码。
    - 大文件输出：如果文件代码行数超过 500 行，请仅输出你修改或新增的完整方法体/函数，不要输出整个文件。
    - 代码块标识：输出的每个代码块必须标注文件路径，例如：// pages/user/index.vue 或 // service/index.ts
    - 修改总结：所有代码输出结束后，用列表形式总结本次修改的文件清单。
